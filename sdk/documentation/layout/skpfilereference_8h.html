LO_RESULT LOGroupClipChildren ( LOGroupRef  group,
LOEntityRef  clip_mask 
)

Applies a clip mask to all entities inside a group. Entities outside the clip mask will be removed from the document. Entities that cross the clip mask boundary will be removed but may be replaced by new entities. The clip_mask entity must be a LORectangleRef, LOEllipseRef, or LOPathRef.

Since
LayOut 2018, API 3.0
Parameters
[in]groupThe group to apply the clipmask to.
[in]clip_maskThe entity to use as a clip mask.
Returns
  • SU_ERROR_NONE on success
  • SU_ERROR_ENTITY_LOCKED if group is locked
  • SU_ERROR_LAYER_LOCKED if group contains entities on locked layers
  • SU_ERROR_GENERIC if clipping algorithm fails

LO_RESULT LOSkpFileReferenceCreate ( LOSkpFileRef *  file_ref,
SUModelRef  model 
)

Creates a SketchUp file reference object with a Skp model. This function takes ownership of the given model. Upon success, user should not release the model, or create another LOSkpFileRef with the same model. The model is valid and mutable till the LOSkpFileRef is released. Do not use a model obtained by a 'Get' function such as the LOSketchUpModelGetModel() as the ownership cannot be transfered to the file_ref.

Since
LayOut 2018, API 3.0
Parameters
[out]file_refThe file reference object to be created.
[in]modelThe SketchUp model.
Returns
  • SU_ERROR_NONE on success
  • SU_ERROR_NULL_POINTER_OUTPUT if file_ref is NULL
  • SU_ERROR_OVERWRITE_VALID if *file_ref refers to a valid object
  • SU_ERROR_INVALID_INPUT if model does not refer to a valid object
